\subsubsection{\nameref{subs:user_interfaces}} % (fold)
Here is a listing of all the requirements concerning the \nameref{subs:user_interfaces}. They are divided into the different applications described in section \ref{ssub:applications} on page \pageref{ssub:applications}.

\paragraph{\nameref{app:mobileuser}} \hfill \\
The user interface of the \nameref{app:mobileuser} has the following requirements:

\begin{enumerate} [label = \textbf{[IR\arabic*]}]

\item When the \emph{\nameref{def:user}} inputs his/her position, the system must output whether a taxi is available or not;

\item When the \emph{\nameref{def:user}} inputs a destination, the system must output whether it is possible to share a ride with another \emph{\nameref{def:user}} or not;

\item When the \emph{\nameref{def:user}} has requested a reservation the system must output either a confirmation or a refusal;

\item When the \emph{\nameref{def:user}} has modified his/her personal information the system must output either a confirmation or a refusal.

\end{enumerate} 

\paragraph{\nameref{app:mobiledriver}} \hfill \\
The user interface of the \nameref{app:mobiledriver} has the following requirements:

\begin{enumerate} [resume*]

\item When the \emph{\nameref{def:taxidriver}} accepts a taxi request the system must output either a confirmation or a refusal;

\item When the \emph{\nameref{def:taxidriver}} logs in the system must show him/her the all the pending requests that he/she can accept;

\item The system must offer a way to the \emph{\nameref{def:taxidriver}} to refresh the list of pending requests that he/she can accept;

\item When the \emph{\nameref{def:taxidriver}} signals any kind of emergency the system must output either a confirmation or a refusal;

\item When the \emph{\nameref{def:taxidriver}} has modified his/her personal information the system must output either a confirmation or a refusal.

\end{enumerate}

\paragraph{\nameref{app:backend}} \hfill \\
The user interface of the \nameref{app:backend} has the following requirements:

\begin{enumerate} [resume*]

\item When the \emph{\nameref{def:operator}} issues a warning to a \emph{\nameref{def:taxidriver}} the system output either a confirmation or a refusal;

\item When the \emph{\nameref{def:operator}} issues a warning to a \emph{\nameref{def:user}} the system output either a confirmation or a refusal.

\end{enumerate}
